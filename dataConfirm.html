Data-Confirm

The data-confirm handler is built to ask the user if they would like to confirm whatever action the button trigers. In this case navigating to another page

The field begins by getting the information required then opens a modal for with confirm and cancel options.

var scope = $('#modal-confirm');
var settings = $(this).data("modal");
var title = $(this).attr("title");
var modal = $('#modal-confirm').show();
var $this = this;

It then sets up listeners to see what action is picked if cancel it closes the modal. If confirm it navigates to the specified href attribute.

$("input[name='cancel']", scope).click(function () {
    modal.hide();
    $event.preventDefault();
});
$("input[name='confirm']", scope).click(function () {
    window.location.href = $this.href;
});

The actual button will look like this.

<a class="btn" data-confirm href='index.html' title="Leave Page" style='margin-bottom:10px'>
navigate to home
</a>

The modal for the confirmation message should be set up like this.

Some of the settings are optional. such as modal-confirm-topbox

<div id="modal-confirm" class="centered" style='display:none'>
    <div id="modal-confirm-topbox"></div>
    <div id="modal-confirm-container">
        <div id="modal-confirm-text">Are you sure you want to navigate away?</div>
            <input name="cancel" type="button" value="Cancel" class="btn btn-danger">
            <input name="confirm" type="button" value="Confirm" class="btn btn-success">
    </div>
</div>
Note: This class is very similar to data-form-confirm and data-form-confirm-post. The differences are simply the action taken on success.
